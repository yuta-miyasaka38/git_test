夜の学習分

  • ワークツリー - ステージ - コミット - github

    リモートレポジトリはgithub上のフォルダ
    （ローカルにコピルと、remote/リモートレポジトリ名/ブランチ名のレポジトリができる）

    リモートレポジトリをpull／fetchするとローカルにブランチができる

    remote origin/master ※リモートブランチの指定は origin/masterでOK

  • git fetch origin ※レポジトリ名originのデータをステージにコピー
  • git merge origin/master ※とってきたレポジトリを現在のブランチにマージ
  • git fetchは2段階、pullは1段階でコピー ※pullは今いるブランチにマージ。ブランチの確認必須
  • git remote ※ローカル内のレポジトリ一覧
  • git remote show origin　※指定したレポジトリの詳細
  • git remote -v ※リモートのURL一覧
  • git remote rename master new_master ※リモート名の変更
  • git remote rm new_master　※リモートの削除

    • ブランチ

  • git branch topic1 ※ブランチ名topic1を作成
  • git log --oneline ※各ブランチがどのコミットを指しているか確認
    HEADの位置も確認 ※自分がいまいるブランチ
  • git branch -b topic1 ※ブランチ作成し、移動
    = git branch topic1 -> git checkout topic1
  • git branch -m topic2　※今いるブランチの名前変更。 -mはmove
  • git branch -d topic2　※ブランチの削除（安全版 mergeされてない場合削除しない）
  • git branch -D topic2 ※ブランチの削除(強制)

    • マージ

  • git maerge リモート名
  • git merge リモート名/ブランチ名
  • fastforward, automerge, conflict
    • fastforward ※進んだ更新に合わせる。ポインタが前に一つ進むだけ
    • automerge　※他のブランチと統合
    • conflict　※同一ファイル変更時に発生。元ファイル修正
